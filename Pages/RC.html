NRF24L01
Remote Controller

A handy circuit to deploy wireless communication in any project.


Background Story

This is remote control circuitry that could elevate any project by adding a wireless communication feature. I made this a couple of years ago to hopefully use in some other projects such as UAV or lawn mower machines.
I was inspired by a YouTube video to create this project however I added extra features to it along the way. The initial objective of this project was to learn how to design a PCB and possibly fabricate the PCB if needed.

Key Features

The transmitter is designed to work with two 18650 LiPo batteries. These batteries are placed in a battery holder and they could power up the board using a two-pin connector and a slide switch. Once the power is inserted through the switch, an LED will turn on to show the presence of power in the circuit.
A 3.3V voltage regulator provides power to the NRF module. The regulator is used to manage the current bursts of the NRF module during send and receive. The other parts of the circuit get power from the batteries directly. The Arduino Nano is located in the middle of the board and controls the behaviour of the circuit. I have used an off-the-shelf Arduino Nano to be able to simply replace or remove it from the pin header if required.
The MPU6050 gyroscope/accelerometer module adds an extra feature to this board to allow the user to control any moving device by tilting the board rather than using the joysticks. The communication protocol of this module is I2C other than that, the NRF module talks to the Arduino via SPI.
Moreover, I added a dedicated circuit on the board that could detect the low voltage state of the batteries and inform the user through a flashing LED. The minimum threshold was adjustable using a trim pot.

Challenges and Solutions

This project was one of my first PCB designs using EasyEDA and I managed to proceed and fabricate the PCB. Below is the image of the PCB that I received from JLCPCB.


This design could get more complex by designing and implementing the Arduino board on my PCB. The modular design of the PCB allows me to create a reliable circuit in a short amount of time rather than involving in the complex design of every module on the board.
To simplify the design and minimise the complexity, I usually create the schematics in blocks so I can separate each section of the design. This strategy also helps me to troubleshoot and understand the circuit quickly. I am also interested in using labels in schematics rather than crossing tracks and wires all around the board which makes it more difficult to analyse the design.

I found out that the naming format was different compared to the product type so I needed to add more features in my application to detect them.
As mentioned before, the user could generate a list of serial numbers and their test results in batches. The application was meant to store all the information on the SQL database however having multiple lists (generated in batches) would create duplicate numbers in the database. I created a new feature to look up the serial number in the SQL database before adding it to the table. The application would simply ignore the duplicated serial numbers and notify the user. Final Words

This project was initiated from a need to have a remote control and add wireless features to my projects. I designed this PCB and was able to get some basic features to work. I developed a simple code in Arduino IDE and test in my lab with a simple receiver circuit on a breadboard however couldn't proceed further due to university constraints at the time.
I didn't have a chance to properly design the receiver PCB but I assume that is quite dependent on the properties and features of the device/machine that needs to be controlled remotely. I hope I will find some time to finish the receiver and test and verify the transmitter and receiver properly.

File Sorting and
Database Update

Sorting more than 70000 files in less than two hours.


Background Story

Analysis and evaluation of the product trends are significantly important to ensure the high quality and reliabilty of the products over the years. This is a project that I did as part of my testing and automation role in the past.
I was assigned to capture test results and identify the number of failed units in the production lines over the last 20 years of a specific product lifecycle. Then the manufacturing process would transfer the failed units to the investigation department so a comprehensive analysis could be performed on the faulty units.
I used LabVIEW to create an application to capture all the test results and folders on the shared drive of the company and categorize them. Other than that, all the test results should have been uploaded to the company's SQL database for further studies on the failed units.

Key Features

The designed application could run automatically and navigate to the folder containing all the required files. Under each folder, there would be at least 5 spreadsheets as test results of 5 different test criteria. These results were labelled as "Passed" or "Failed" so my application could identify them by their file naming format.
The LabVIEW application would create a master spreadsheet and export all the product serial numbers along with their test results. Other than that, the reliability of the test stations could be evaluated because I could identify the test stations by analyzing the test results and their parameters.
This application was mainly designed to automatically run every day without any human interactions so I did not spend much time designing the front panel and instead focused on the performance and reliability of the application.
Due to the large number of test results, I added an option to the application to allow the user to export test results within a specific timeframe. The user could set the first and end dates to evaluate the test results. This could reduce the run time and allow the user to capture the results in batches.

Challenges and Solutions

This project was one of the most challenging projects that I had ever done. There was so much complexity in reading and categorizing the folders and test files.
I needed to identify which units were sent to the investigation department. Both test systems in the production floor and investigation department would generate test reports in the same format. I realized that I could only detect the investigation units by the properties of the file. I managed to extract the owner of the files using the CMD function of Windows so I could detect the reports generated by the computers in the investigation department.
The below screenshot shows how the CMD command runs in the background of the application. The below snapshot shows the LabVIEW code to do so.


I found out that the naming format was different compared to the product type so I needed to add more features in my application to detect them.
As mentioned before, the user could generate a list of serial numbers and their test results in batches. The application was meant to store all the information on the SQL database however having multiple lists (generated in batches) would create duplicate numbers in the database. I created a new feature to look up the serial number in the SQL database before adding it to the table. The application would simply ignore the duplicated serial numbers and notify the user. Final Words

This project was one of the few LabVIEW applications that I created from scratch. This application simplifies the process of product change analysis over the years.
Eventually, I locked the application to generate one spreadsheet for one day only. The management computer would turn on every day and my application was configured to run automatically every day and capture all the results for the day before and upload them to the SQL database.
This project was further developed by the company's software/database engineer to create a graphical summary (using Power BI) of test results per day and month.
